REST API routes Auth¶

“Module routs.auth

async routes.auth.confirmed_email(token: str, db: Session = Depends(get_db))¶
The confirmed_email function is used to confirm a user’s email address.

It takes the token from the URL and uses it to get the user’s email address. The function then checks if there is a user with that email in our database, and if not, returns an error message. If there is such a user, we check whether their account has already been confirmed or not; if it has been confirmed already, we return an appropriate message; otherwise we update their account status in our database.

Parameters:

  • token – str: Get the token from the url

  • db – Session: Pass the database session to the function

Returns:

A message if the email is already confirmed

It takes the email and password of the user as input, and returns an access token if authentication was successful.

async routes.auth.refresh_token(credentials: HTTPAuthorizationCredentials = Security(HTTPBearer), db: Session = Depends(get_db))¶
The refresh_token function is used to refresh the access token.

The function takes in a refresh token and returns an access_token, a new refresh_token, and the type of token. If there is no valid user associated with the given email address or if there are any errors during decoding or creating tokens then an HTTPException will be raised.

Parameters:

  • credentials – HTTPAuthorizationCredentials: Get the token from the request header

  • db – Session: Get the database session

Returns:

A dictionary with the access_token, refresh_token and token type

async routes.auth.request_email(body: RequestEmail, background_tasks: BackgroundTasks, request: Request, db: Session = Depends(get_db))¶

The request_email function is used to send an email to the user with a link that will allow them to confirm their email address. The function takes in a RequestEmail object, which contains the email of the user who wants to confirm their account. It then checks if there is already a confirmed account associated with that email address, and if so returns an error message saying as much. If not, it sends an email containing a confirmation link.

Parameters:

  • body – RequestEmail: Validate the request body

  • background_tasks – BackgroundTasks: Add tasks to the background task queue

  • request – Request: Get the base url of the application

  • db – Session: Access the database

Returns:

A message that depends on the user’s email status

async routes.auth.signup(body: UserAuthModel, background_tasks: BackgroundTasks, request: Request, db: Session = Depends(get_db))¶
The signup function creates a new user in the database.

It takes in a UserAuthModel object, which contains an email and password. The function checks if the email already exists, and if it does not exist, then it hashes the password using bcrypt and adds that user to the database.

Parameters:

  • body – UserAuthModel: Get the user’s email and password

  • background_tasks – BackgroundTasks: Add a task to the background tasks queue

  • request – Request: Get the base_url of the server

  • db – Session: Access to database

Returns:

A user object

async routes.users.update_avatar_user(file: UploadFile = File(PydanticUndefined), current_user: User = Depends(get_current_user), db: Session = Depends(get_db))¶

The update_avatar_user function takes in a file, the current user and the database session. It then uploads the file to cloudinary with a public id of UsersPhoto/username. It overwrites any existing files with that name. The function then builds an url for that image using CloudinaryImage and returns it as src_url. The function finally updates the avatar field in users table for this user’s email address.

Parameters:

  • file – UploadFile: Upload the file to cloudinary

  • current_user – User: Get the current user from the database

  • db – Session: Get the database session

Returns:

The user object, which is then sent to the client

async services.email.send_email(email: EmailStr, username: str, host: str)¶
The send_email function sends an email to the user with a link to confirm their email address.
The function takes in three parameters:

-email: EmailStr, the user’s email address. -username: str, the username of the user who is registering for an account. This will be used in a greeting message within the body of the email sent to them. -host: str, this is where we are hosting our application (i.e., localhost). This will be used as part of a URL that they can click on within their browser.

  • username – str: Pass the username of the user to be used in the email template
